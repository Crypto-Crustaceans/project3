About

For our final project, our goal is to analyze the effect of financial news headlines along with transaction volume on Bitcoin and display the result in an easy to intpetret dashboard. We are incorporating market data and news articles from the CryptoCompare API. For future news articles we will be using a scrapper for bitcoinnews.com and will use FinBert for sentiment analysis. The sentiment analysis will be used to project future price projections based off the previous 7 days volume and news sentiment.

FinBERT Sentiment Analysis

FinBERT's engine is a pre-trained sentiment analysis model that analyzes sentiment of financial text. It is built by further training the BERT language model in the finance domain. We accessed a large financial corpus to analyze close to 4,000 news articles.

CryptoCompare

We've extracted blockchain data, such as transaction volume and price, using the CryptoCompare API.

JavaScript Visualizations

We leveraged two JS libraries to build our visualizations. D3.js combines powerful visualization components and a data-driven approach to DOM manipulation to create custom, dynamic, and interactive data visualizations. Chart.js is used to simple, clean, and engaging HTML5 based JavaScript charts.

Project Timeline

Our team built on the skills and process learned during our previous projects and asked ourselves "If we had more time then, what else would we work on?" It didn't take long for all of us to agree that the wealth of information we scraped and analyzed could have been presented in a much more exciting way. With all we've learned in and out of class since then, we've decided to explore different ways to analyze and display our data. Here's how we did it.

  • Extract:

    CryptoCompare and FinBERT Sentiment Analysis

    We utilized CryptoCompare API to pull blockchain data for Bitcoin. The API allowed us to pull very specific data from a large timeframe. The scraped data records the date and time, open, high, low, close, and volume traded per day. We first exported the raw data into a CSV format in order to manipulate and test the data.

    We also pulled ~4,000 news articles about Bitcoin that goes back to 2016 to run sentiment analysis with a pre-trained model FinBERT. This model built on BERT, a machine learning technique for natural language processing, and focused on financial languages. This mean we could load in our news articles and the model would loop through each one and calculate a score to determine if the article is positive, neutral, or negative. The model had to be trained first by downloading a phrasebank online. We then set up model training in a Jupyter notebook, where we've set up training parameters. Once everything was set up, we then ran the analysis through all the news article and exported the results in a CSV file.

  • Transform:

    Data Clean Up & Analysis

    The most effective way to manipulate our data is PostgreSQL. We put together the SQL schemas using QuickDBD diagram tool. The three tables were then loaded: BTC market data, daily news articles, and the sentiment analysis. We ran a SQL query to join all three in an easy to understand way that our JavaScript could read and plot.

    In order to allow the three tables to "speak" we had to first manipulate the formattng of date and time in the data from the news articless. We first merged the two separate date and time columns into one datetime column, then account for the extra hour during Daylight Saving Time. Finally, we adjusted the string to match the format CryptoCompare scraper pulled. Once the data is clean, they are exported into a JSON format.

  • Load:

    Putting Everything Together with SQLAlchemy and Flask

    We created a Flask application to serve up data and power our JavaScript visualizations. The Flask app queries data from our PostgreSQL database and converts it into JSON format. The JSON can be accessed at the '/data' route by our JavaScript code. The '/home' route accesses our HTML page visualizations.

  • Visualization:

    Plotting with JavaScript and Sending the Project Live

    There is one visualtization our group decided on for this dataset. The first is a financial calendar that shows BTC movement for the day using a colorscale from green to red along with our model's prediction for the day. Green represents the market closing higher than prior day, and red represents the opposite. The two triangles connect to form a square that represents a day.
